# OpenLogProbs

**Section:** 7.9 — Practical Tools and Infrastructure

**Authors:** Justin Chiu et al.

**Code:** [github.com/justinchiu/openlogprobs](https://github.com/justinchiu/openlogprobs)

---

## Why This Paper Is Relevant

OpenLogProbs solves a critical practical problem: **extracting full next-token distributions from APIs that only expose top-k logprobs.** This expands the range of models our DAL can work with.

---

## Key Concept: Logit Bias Bisection

APIs like OpenAI expose only top-5 logprobs. OpenLogProbs uses a clever trick:
1. Apply **logit bias** to suppress the known top-k tokens
2. Re-query the API to reveal the next set of tokens
3. Repeat (bisection search) to reconstruct the full distribution

This allows full vocabulary probability estimation from APIs that only expose partial logprobs.

---

## How We Use It in Distributional GEPA

1. **Extends model coverage:** The DAL's primary limitation is logprob access. OpenLogProbs partially addresses this for OpenAI models (top-5 → full distribution reconstruction).

2. **Cost consideration:** Each bisection step requires an additional API call. For GEPA's budget-constrained setting, the cost may be prohibitive for routine use but acceptable for targeted diagnostics.

3. **Not needed for primary experiments:** Our primary experiments use open-weight models via vLLM (full vocabulary access). OpenLogProbs is relevant for secondary experiments with GPT-4o.

## Connection to Other Papers

- Addresses the model access constraint discussed in **Section 2.3** of PROJECT_SEED
- Complements **LM-Polygraph (7.9.1)** and **LogitScope (7.9.2)** which compute metrics from logprobs but don't solve the access problem
- Relevant to the **Logprobs API Availability table** (Section 7.10): OpenLogProbs upgrades "top-5" providers to "full vocabulary" (at a cost)

---

## Practical Limitations

- Multiple API calls per token position = expensive
- Adds latency per example
- Only works with APIs that support logit bias (OpenAI, some others)
- Cannot help with Anthropic Claude (no logprobs at all, no logit bias)
