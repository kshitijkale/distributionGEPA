# The Anatomy of Uncertainty in LLMs (Three-Way Decomposition)

**Section:** 7.7 — Uncertainty Decomposition

**Authors:** (2026)

**Venue:** 2026

---

## Why This Paper Is Relevant

Proposes a three-way decomposition of LLM uncertainty that goes beyond the standard aleatoric/epistemic split. The three sources — input ambiguity, knowledge gaps, and decoding randomness — map to qualitatively different distributional signatures, informing why different signatures might demand different prompt mutations.

---

## Three-Way Decomposition

| Source | Description | Distributional Signature | Prompt Intervention |
|--------|-------------|------------------------|---------------------|
| **Input ambiguity** | The question is genuinely ambiguous | Multiple semantically coherent but different answer clusters | Add disambiguation or clarification instructions |
| **Knowledge gaps** | The model lacks relevant knowledge | Diffuse distribution, no clear preference | Add factual context, retrieval instructions, or domain knowledge |
| **Decoding randomness** | Sampling noise produces different answers | Answers cluster tightly with minor phrasing variations | Temperature adjustment or self-consistency (not a prompt issue) |

---

## How We Use It in Distributional GEPA

1. **Framework for interpreting distributional patterns:** When the DAL detects uncertainty, this three-way framework helps interpret *why*:
   - Bimodal forking token → likely input ambiguity
   - Diffuse forking token → likely knowledge gap
   - Low entropy with occasional outlier samples → likely decoding randomness

2. **Informs H4 (emergent differentiation):** We hypothesize the reflector will spontaneously produce different mutations for different uncertainty sources. This framework provides categories for analyzing whether that happens.

3. **Filtering non-actionable uncertainty:** Decoding randomness is not a prompt issue — identifying it helps the reflector avoid wasting mutations on problems that prompt changes can't fix.
