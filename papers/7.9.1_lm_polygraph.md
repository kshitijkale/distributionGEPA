# LM-Polygraph: Uncertainty Estimation for Language Models

**Section:** 7.9 — Practical Tools and Infrastructure

**Authors:** Fadeeva et al.

**Venue:** EMNLP 2023 (Demo), TACL 2025 (Benchmark)

**Code:** [github.com/IINemo/lm-polygraph](https://github.com/IINemo/lm-polygraph)

---

## Why This Paper Is Relevant

LM-Polygraph is a **unified API for 15+ uncertainty estimation methods** — the primary candidate for our uncertainty computation backend. Instead of implementing entropy computation, semantic entropy, and other methods from scratch, we can build on this existing infrastructure.

---

## Key Features

1. **15+ uncertainty methods:** Token-level entropy, sequence-level perplexity, semantic entropy, P(True), Monte Carlo dropout, and more — all behind a unified API.

2. **Multiple model backends:** Supports HuggingFace models, OpenAI API, and other providers.

3. **Benchmark suite:** Standardized evaluation of uncertainty methods across tasks and models.

---

## How We Use It in Distributional GEPA

1. **Primary uncertainty computation backend:** Rather than implementing entropy computation from scratch, use LM-Polygraph's existing implementations for:
   - Token-level Shannon entropy
   - Sequence-level perplexity
   - Semantic entropy (if self-consistency variant is used)

2. **Method comparison:** LM-Polygraph's unified API makes it easy to swap between uncertainty methods in ablations. We can test which method provides the best DAL features without reimplementing each method.

3. **Integration path:** LM-Polygraph → compute diagnostics → DAL annotation → GEPA reflective dataset.

---

## Practical Considerations

- Need to verify LM-Polygraph supports vLLM backends (our primary experimental setup)
- May need custom extensions for forking token identification and entropy trajectory computation
- The library may add overhead vs. direct logprob processing — benchmark before committing
