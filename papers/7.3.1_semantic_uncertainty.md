# Semantic Uncertainty: Linguistic Invariances for Uncertainty Estimation in Natural Language Generation

**Section:** 7.3 — Semantic Entropy and Meaning-Level Uncertainty

**Authors:** Lorenz Kuhn, Yarin Gal, Sebastian Farquhar (Oxford OATML)

**Paper:** [arXiv:2302.09664](https://arxiv.org/abs/2302.09664)

**Venue:** ICLR 2023

---

## Why This Paper Is Essential

This paper introduces **semantic entropy** — a foundational concept that addresses a critical limitation of token-level entropy: conflating meaning uncertainty with phrasing uncertainty. It provides the conceptual foundation for how our DAL should interpret distributional signals.

---

## Key Concept: Semantic Entropy

**Problem with token-level entropy:** A model might assign high entropy to a token position because it's uncertain between "Paris" and "London" (meaning uncertainty — genuinely doesn't know the answer) or because it's uncertain between "The answer is Paris" and "Paris is the answer" (phrasing uncertainty — knows the answer but not how to phrase it). Token-level entropy conflates these two fundamentally different situations.

**Solution — Semantic Entropy:**
1. Sample multiple complete outputs from the model
2. Cluster outputs by **meaning** using bidirectional natural language inference (NLI): two outputs are in the same cluster if they entail each other
3. Compute entropy over **semantic clusters**, not individual tokens/sequences

```
SE = -Σ P(cluster_i) log P(cluster_i)
```

Where cluster probabilities are estimated from sample frequencies.

**Result:** High semantic entropy = the model genuinely doesn't know the answer. Low semantic entropy = the model knows the answer (even if phrasing varies).

---

## How We Use It in Distributional GEPA

1. **Conceptual foundation:** Our DAL occupies a **middle ground** between raw token-level entropy and full semantic entropy:
   - We use **token-level distributional features** (entropy, competing tokens) for computational efficiency
   - But we annotate them with **semantic context** (what competing tokens *mean*, not just their probabilities)
   - "The model was torn between 'Paris' and 'London'" is a semantic annotation of a token-level distributional fact

2. **Design principle:** When the DAL identifies forking tokens, reporting the **identities** of competing tokens provides semantic information that raw entropy values don't. Knowing the model hesitated between "Paris" and "London" (different cities — meaning uncertainty) is more diagnostic than knowing it hesitated between "Paris" and "paris" (capitalization — phrasing uncertainty).

3. **Limitation awareness:** Our approach doesn't fully solve the phrasing vs. meaning confusion that semantic entropy addresses. We acknowledge this and test whether token-level features with semantic annotation are "good enough" for the reflector.

## Key Numbers

- Requires multiple samples (typically 5-10) for reliable cluster estimation
- Significantly better than token-level probability for hallucination detection
- Computationally expensive: N forward passes + NLI classification for all pairs

---

## Implications for Design

- **Self-consistency (Condition D / text-enriched baseline):** Self-consistency can be viewed as a cheap approximation to semantic entropy. If Condition D (which uses self-consistency) matches Condition E (which uses logprob-based features), it may be because self-consistency captures most of the semantic uncertainty signal.
- **Cost trade-off:** Full semantic entropy is too expensive for GEPA's budget-constrained setting. Our token-level + semantic annotation approach is a practical compromise.
- **The text-enriched baseline threat:** This paper's insight — that meaning-level uncertainty matters more than token-level uncertainty — suggests that text-based methods (self-consistency, verbalized confidence) might capture the right signal without needing logprobs. This strengthens Condition D as a baseline.
